네트워크에 대한 이해
--
__TCP/IP 의 동작 이해__
- TCP/IP
  - TCP/IP 는 TCP 프로토콜과 IP 프로토콜을 아울러 지칭하는 용어
  - TCP 프로토콜 : 신뢰성 있고 무결성을 보장하는 연결을 통해 데이터를 안전하게 전달해주는 전송 프로토콜
  - IP 프로토콜 : 패킷들을 가장 효율적인 방법으로 최종 목적지로 전송하기 위해 필요한 프로토콜
  
- TCP/IP 4계층
  - 4계층 : 응용 계층 (Application)
  - 3계층 : 전송 계층 (Transport)
  - 2계층 : 인터넷 계층 (Internet)
  - 1계층 : 네트워크 인터페이스 계층 (Network Interface)

```
TCP/IP 프로토콜을 따르는 네트워크 통신이라면, 송신 호스트에서 전송할 데이터를 TCP/IP 4계층이라는 계층 구조에 따라 순차적으로
캡슐화를 하여 네트워크에 전송하게 된다. 그리고 데이터를 수신하는 호스트에서도 이 계층 구조에 따라 캡슐화된 패킷을 역순으로 포장을
뜯어가며 원래 데이터를 확인해볼 수 있게 되는 것이다.
```

__3-Hand Shaking 에 대한 이해__
- TCP 통신을 이용하여 데이터를 전송하기 위해 네트워크 연결을 설정(Connection Establish) 하는 과정
- 작동 방식
  - SYN(Synchronization) : 연결요청, 세션을 설정하는데 사용되며 초기에 시퀀스 번호를 보냄
  - ACK(Acknowledgement) : 보낸 시퀀스 번호에 TCP 계층에서의 길이 또는 양을 더한 것과 같은 값을 ACK에 포함하여 전송
      - Step 1 (SYN) :클라이언트는 서버와 커넥션을 연결하기 위해 SYN을 보낸다. (seq : x)
      - Step 2 (SYN + ACK) : 서버가 SYN(x)을 받고, 클라이언트로 받았다는 신호인 ACK와 SYN 패킷을 보냄 (seq : y, ACK : x + 1)
      - Step 3 (ACK) : 클라이언트는 서버의 응답은 ACK(x+1)와 SYN(y) 패킷을 받고, ACK(y+1)를 서버로 보냄

URL 구조 파악
--
- clients가 서버로부터 특정 리소르를 요청할 수 있도록 하려면, clients가 서버에 파일을 요청하는 방법을 알아야 하는 이름 지정 메커니즘이 필요하다.
이러한 이름 지정 메커니즘이 __Uniform Resource Locator (URL)__ 이다.
- Protocol, Domain, Path, Query String, Fragement 로 구성
  - ex) http://www.funwebdev.com/index.php?page=17#article
    
개발자 도구 정보 파악
--
__크롬 개발자 도구__
- 개발자 도구 열기
  - 단축키 - Window + Shift + c
  - 마우스 - 화면에 오른쪽 클릭 후에 아래 보이는 검사(inspect) 클릭
- 요소 활용하기
  1. 페이지 내의 특정 요소 지정
      - 개발자 도구 좌측 상단의 첫번째 버튼을 누르면 특정 요소를 지정 할 수 있음.
  2. 페이지 전체의 요소 파악하기
      - 메뉴 탭의 'Element'를 클릭하면 해당 페이지의 html 문서 확인 가능.
      - html 문서의 원하는 라인에 마우스를 올려두면 해당 요소의 값과 레이아웃 확인 가능
- 콘솔 활용법
  1. 로그 확인하기
  2. 데이터 확인하기
  3. 에러 확인하기 

서버와 클라이언트에 대한 이해
--
- 클라이언트(Client)
  - 서버의 서비스를 받아 사용하는 장치, 프로그램
  - 클라이언트 장치 : 최종 사용자가 웹에 접속하는데 사용하는 시스템 ex) 데스크탑, 노트북, 스마트폰, 태블릿
  - 클라이언트 프로그램 : 사용자가 웹을 통해 요청할 수 있게 해주는 프로그램 ex) 웹 브라우저
 
- 서버(Server)
  -  네트워크를 통해 클라이언트에게 서비스를 제공하는 시스템
    
HTTP 요청, 응답 메시지에 대한 이해
--
__HTTP__
- http 프로토콜은 기본적으로 포트 80에서 TCP 연결을 설정한다. (HTTPS의 경우 443)
- 요청-응답 방식으로 작동한다.
  - 서버는 요청을 기다리고, 그런 다음 응답 코드와 헤더 및 선택적인 메시지(파일을 포함할 수 있음)로 응답함.
